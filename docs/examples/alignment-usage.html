Alignment Usage

This section provides some alignment examples. Alignment means in this case that additional value alignment instructions are used as well. The alignment itself is separated by comma and is followed by the total character width. A negative value means “left justified” and a positive value means “right justified”. But not a plus sign '+' for a positive value is not allowed.

Index–based Formatting

This example shows an index–based formatting with system types as well as with value alignment.

Copy
Template.Format("left [{1,-20}] right [{0,20}]", 111111, 222222);

The result would look like "left [222222 ] right [ 111111]".

This example shows an index–based formatting with system types as well as with value alignment and with format instructions.

Copy
Template.Format("left [{1,-20:N2}] right [{0,20:C3}]", 111111, 222222);

The result would look like "left [222,222.00 ] right [ $111,111.000]".

Label–based Formatting

This example shows a label–based formatting with system types as well as with value alignment.

Copy
Template.Format("left [{ValA,-20}] right [{ValB,20}]", 111111, 222222);

The result would look like "left [111111 ] right [ 222222]". This is pretty much the same result as for its index–based counterpart except that the argument order is different.

This example shows a label–based formatting with system types as well as with value alignment and with format instructions.

Copy
Template.Format("left [{ValA,-20:N2}] right [{ValB,20:C3}]", 111111, 222222);

The result would look like "left [111,111.00 ] right [ $222,222.000]". This is pretty much the same result as for its index–based counterpart except that the argument order is different.
