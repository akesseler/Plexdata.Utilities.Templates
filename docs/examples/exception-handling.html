Exception Handling

By design, the Plexdata Template Formatter does not handle exceptions internally, except for value formatting. Hence, the implementation checks and validates each of its input parameters so that an unexpected exception becomes unlikely to occur. But there is no guarantee that exceptions will never be thrown.

Sometimes value formatting may fail and usually ends up in an exception. As mentioned above, the template formatter is designed to suppress exceptions as often as possible. But exceptions that occur during value formatting are caught and will be put into the output, instead of re–throwing them. Here an example.

The code snippet below uses a wrong date format which in turn causes a format exception.

Copy
Decimal price = 123456.789M;
DateTime date = DateTime.Parse("2020-10-29T23:05:17.123");

Template.Format("Price: {price:C2} Date {date:K}", price, date);

Instead of re–throwing the format exception, the result would look like "Price: $123,456.79 Date [{date:K} => FormatException: "Input string was not in a correct format."]".

Keep in mind, this behaviour also includes exceptions that might be thrown in any implementation of interface IArgumentSerializer.
