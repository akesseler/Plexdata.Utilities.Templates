Extended Formatting

This section provides some extended examples. Extended means in this case that additional value format instructions are used as well.

Index–based Formatting

This example shows an index–based formatting with system types as well as with format instructions.

Copy
Template.Format("text {1:N2} text {0:C} text {2}", 1111111.1111, 2222.2222, 333.33);

The result would look like "text 2,222.22 text $1,111,111.11 text 333.33". This should be expected because of its the same result as for String.Format().

This example shows an index–based formatting with system types and format instructions as well as with stringification. In other words, the operator '$' is used together with argument indices.

Copy
Template.Format("text {$1:N2} text {$0:C} text {$2}", 1111111.1111, 2222.2222, 333.33);

The result would look like "text 2222.2222 text 1111111.1111 text 333.33". The reason is method ToString() of each argument is called instead of performing argument formatting.

This example shows an index–based formatting with system types and format instructions as well as with serialization. In other words, the operator '@' is used together with argument indices.

Copy
Template.Format("text {@1:N2} text {@0:C} text {@2}", 1111111.1111, 2222.2222, 333.33);

The result would look like "text 2,222.22 text $1,111,111.11 text 333.33". Ends up with the same result as for standard formatting.

Label–based Formatting

This example shows a label–based formatting with system types as well as with format instructions.

Copy
Template.Format("text {ValA:N2} text {ValB:C} text {ValC}", 1111111.1111, 2222.2222, 333.33);

The result would look like "text 1,111,111.11 text $2,222.22 text 333.33". This should be expected because of its the same result as for String.Format().

This example shows a label–based formatting with system types and format instructions as well as with stringification. In other words, the operator '$' is used together with template labels.

Copy
Template.Format("text {$ValA:N2} text {$ValB:C} text {$ValC}", 1111111.1111, 2222.2222, 333.33);

The result would look like "text 1111111.1111 text 2222.2222 text 333.33". The reason is method ToString() of each argument is called instead of performing argument formatting.

This example shows a label–based formatting with system types and format instructions as well as with serialization. In other words, the operator '@' is used together with template labels.

Copy
Template.Format("text {@ValA:N2} text {@ValB:C} text {@ValC}", 1111111.1111, 2222.2222, 333.33);

The result would look like "text 1,111,111.11 text $2,222.22 text 333.33". Ends up with the same result as for standard formatting.
