Simple Formatting

This section provides really simple examples. In other words, only the basics of the formatting are shown, but with the aim of getting a first impression.

Index–based Formatting

This example shows an index–based formatting with system types.

Copy
Template.Format("text {1} text {0} text {2}", 111, "ABC", 333.33);

The result would look like "text ABC text 111 text 333.33".

This example shows the same as above but with one missing argument.

Copy
Template.Format("text {1} text {0} text {2}", 111, "ABC");

The result would look like "text ABC text 111 text {2}". Please note, in such a case the whole formatting instruction is inserted instead of some value. See section Fallback Usage for an example of how to avoid this.

This example shows a special feature, an index–based formatting with system types and an inline argument array.

Copy
Template.Format("text {1} text {0} text {2} text {3} text {2}", "ABC", new[] { 111, 222, 333 });

The result would look like "text 111 text ABC text 222 text 333 text 222".

This example shows an index–based formatting with system types as well as with stringification. In other words, the operator '$' is used together with argument indices.

Copy
Template.Format("text {$1} text {$0} text {$2}", 111, "ABC", 333.33);

The result would look like "text ABC text 111 text 333.33". For sure, stringification does not really make sense for system types. But it is interesting for complex data types, because of any formatting is skipped and ToString() is called instead.

This example shows an index–based formatting with system types as well as with serialization. In other words, the operator '@' is used together with argument indices.

Copy
Template.Format("text {@1} text {@0} text {@2}", 111, "ABC", 333.33);

The result would look like "text ABC text 111 text 333.33". For sure, serialization does not really make sense for system types. But it is interesting for complex data types, because of any formatting is skipped and the serializer (custom or build–in) is called instead.

Label–based Formatting

This example shows a label–based formatting with system types.

Copy
Template.Format("text {ValA} text {ValB} text {ValC}", 111, "ABC", 333.33);

The result would look like "text 111 text ABC text 333.33". This is pretty much the same result as for its index–based counterpart except that the argument order is different. Keep in mind, label–based formatting always resolves the arguments from left to right.

This example shows the same as above but with one missing argument.

Copy
Template.Format("text {ValA} text {ValB} text {ValC}", 111, "ABC");

The result would look like "text 111 text ABC text {ValC}". Please note, in such a case the whole formatting instruction is inserted instead of some value. See section Fallback Usage for an example of how to avoid this.

This example shows a special feature, a label–based formatting with system types and an inline argument array.

Copy
Template.Format("text {ValA} text {ValB} text {ValC} text {ValD}", "ABC", new[] { 111, 222, 333 });

The result would look like "text ABC text 111 text 222 text 333".

This example shows a label–based formatting with system types as well as with stringification. In other words, the operator '$' is used together with template labels.

Copy
Template.Format("text {$ValA} text {$ValB} text {$ValC}", 111, "ABC", 333.33);

The result would look like "text 111 text ABC text 333.33". For sure, stringification does not really make sense for system types. But it is interesting for complex data types, because of any formatting is skipped and ToString() is called instead.

This example shows a label–based formatting with system types as well as with serialization. In other words, the operator '@' is used together with template labels.

Copy
Template.Format("text {@ValA} text {@ValB} text {@Valc}", 111, "ABC", 333.33);

The result would look like "text 111 text ABC text 333.33". For sure, serialization does not really make sense for system types. But it is interesting for complex data types, because of any formatting is skipped and the serializer (custom or build–in) is called instead.
